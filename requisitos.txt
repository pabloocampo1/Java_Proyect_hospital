Requisitos para el Proyecto de Gestión de Pacientes y Doctores
Requisitos Funcionales

solo las clases - Gestión de Doctores:

Agregar Doctor: La aplicación debe permitir agregar nuevos doctores con atributos como nombre, edad, especialidad y ID único.
Eliminar Doctor: La aplicación debe permitir eliminar doctores existentes utilizando su ID.
Actualizar Información del Doctor: La aplicación debe permitir actualizar la información de los doctores.
Listar Doctores: La aplicación debe permitir listar todos los doctores registrados.
Buscar Doctor: La aplicación debe permitir buscar un doctor por su nombre o especialidad.

Gestión de Pacientes:

Agregar Paciente: La aplicación debe permitir agregar nuevos pacientes con atributos como nombre, edad, ID único y médico asignado.
Eliminar Paciente: La aplicación debe permitir eliminar pacientes existentes utilizando su ID.
Actualizar Información del Paciente: La aplicación debe permitir actualizar la información de los pacientes.
Listar Pacientes: La aplicación debe permitir listar todos los pacientes registrados.
Buscar Paciente: La aplicación debe permitir buscar un paciente por su nombre o por el nombre del médico asignado.

Asignación de Doctores a Pacientes:

Asignar Doctor a Paciente: La aplicación debe permitir asignar un doctor a un paciente.
Cambiar Doctor Asignado: La aplicación debe permitir cambiar el doctor asignado a un paciente.
Listar Pacientes por Doctor: La aplicación debe permitir listar todos los pacientes asignados a un doctor específico.
Historial Médico:

Agregar Registro Médico: La aplicación debe permitir agregar registros médicos a los pacientes.
Ver Historial Médico: La aplicación debe permitir ver el historial médico completo de un paciente.
Actualizar Registro Médico: La aplicación debe permitir actualizar registros médicos existentes.
Autenticación y Autorización:

Autenticación de Usuarios: La aplicación debe requerir autenticación para acceder a las funcionalidades administrativas.
Roles de Usuario: La aplicación debe permitir la creación de diferentes roles de usuario (e.g., Administrador, Doctor, Recepcionista) y restringir el acceso a ciertas funcionalidades según el rol.
